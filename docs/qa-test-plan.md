# QA/테스트 전략 — plan2space

문서 목적: 도면 분석 정확도와 3D/워크스루 UX 품질을 안정적으로 유지하기 위한 테스트 범위와 릴리즈 게이트를 정의한다.

---

## 1) 테스트 원칙
- 자동화는 **도면 파이프라인/3D 생성**에 집중
- 입력 다양성(해상도/선 두께/스케일)을 대표하는 샘플셋 회귀
- 성능 저하는 UX 손실로 직결되므로 FPS/로딩 시간 체크 포함

---

## 2) 테스트 레벨

- Unit: Plan2D 검증, 좌표/치수 변환, 벽/오프닝 생성 로직
- Integration: 도면 업로드 → AI 분석 → 2D 보정 → 3D 생성
- E2E: 업로드부터 저장/불러오기까지 사용자 플로우
- Performance: 중간 규모 씬 기준 FPS 유지 여부

---

## 3) 핵심 시나리오

### 3.1 도면 → 3D 파이프라인
- 이미지 업로드 → JSON 생성 → 2D 수정 → 3D 렌더
- 실패 시 재시도/보정 UI 복구 가능

### 3.2 2D 보정 UX
- 벽/문/창 드래그 및 스냅
- 닫히지 않은 폴리곤 오류 표시

### 3.3 1인칭 워크스루
- WASD + 포인터 락 이동
- 벽/가구 충돌 방지
- 문 클릭 인터랙션 정상 동작

### 3.4 저장/불러오기
- 씬 저장 후 재로드 시 상태 일치
- 스냅샷 생성 및 썸네일 표시

### 3.5 인증/권한
- 로그인 사용자만 프로젝트 접근 가능
- 다른 사용자 프로젝트 접근 차단(RLS)

---

## 4) 품질 게이트
- 핵심 E2E 시나리오 통과
- 기본 기기 기준 FPS 목표 충족
- 도면 샘플셋 회귀 결과 유지
