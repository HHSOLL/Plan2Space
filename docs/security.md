# 보안 요구사항 — plan2space

문서 목적: 도면 업로드/AI 처리/프로젝트 저장 과정에서 필요한 최소 보안 기준을 정의한다.

---

## 1) 보호 대상
- 사용자 업로드 도면 이미지
- 프로젝트(씬) JSON + 스냅샷
- API 키/환경 변수

---

## 2) 주요 위협과 대응

### 2.1 파일 업로드
- 대용량/비정상 파일
- 대응: 확장자/MIME 제한, 파일 크기 제한

### 2.2 API 키 유출
- 대응: 서버 전용 환경 변수 사용, 클라이언트 노출 금지

### 2.3 권한 우회
- 다른 사용자의 프로젝트 접근
- 대응: 서버 측 소유자 검증, 공유 링크 만료

---

## 3) 최소 보안 요구사항

- 업로드 파일 크기 제한
- AI 호출은 서버에서만 수행
- 저장/불러오기 시 사용자 소유권 검증
- RLS 정책으로 프로젝트 접근 제한
