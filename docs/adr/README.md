# ADR(Architecture Decision Record)

문서 목적: 아키텍처/정책의 중요한 결정을 “왜 그렇게 했는지” 기록해, 팀 변경/확장 시에도 일관성을 유지한다.

## 규칙
- 파일명: `NNN-title.md` (예: `001-auth-scheme.md`)
- 각 ADR은
  - 상태: `proposed | accepted | superseded`
  - 배경/문제, 고려한 대안, 결정, 결과(트레이드오프) 포함
- PR에 ADR 링크를 포함한다.
